# productdebt.md — Product Improvement Register

**Project:** _TBD_   **Maintained by:** Claude Code (and founder)

> The product-side twin of `techdebt.md`. Where `techdebt.md` records CODE issues to fix,
> `productdebt.md` records PRODUCT ideas — improvements, enhancements, and feature thoughts that occur
> DURING development but are out of scope for the current phase. Capturing them here keeps the current
> build focused while making sure good ideas are never lost.

🟦 **FOR CLAUDE CODE** — whenever a product improvement or enhancement idea surfaces mid-build (yours
or the founder's), log it here instead of acting on it or letting it derail the current phase. Review
this list at phase boundaries to decide what graduates into `implementation.md` as planned work.

🟩 **FOR THE FOUNDER** — this is the "good idea, not right now" parking lot. It protects the current
phase from scope creep while guaranteeing the idea is recorded for a future phase.
