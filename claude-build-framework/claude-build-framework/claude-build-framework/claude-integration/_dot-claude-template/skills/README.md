# skills/ — project-specific capabilities, loaded on demand

Add skills only as the project needs them (keeps context light). Each skill is a folder with a SKILL.md
describing when it triggers and what it does. Decide which skills a project needs at planning
(Section 9 Step 3) and record them in TRD.md. Examples a project MIGHT need: PDF generation, data
import, a specific API workflow. Do not add skills speculatively.
